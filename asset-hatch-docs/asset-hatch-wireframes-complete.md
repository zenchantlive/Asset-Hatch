## Accessibility Considerations

### Screen Readers
- All images have alt text
- Chat messages have `role="log"`
- User vs Agent messages distinguished by `aria-label`
- Form inputs have associated `<label>` elements
- Buttons have descriptive text (not just icons)

### Keyboard Navigation
- Tab order: Header â†’ Main controls â†’ Content â†’ Footer
- Modals trap focus
- Escape closes modals
- Enter submits forms
- Arrow keys navigate through options where appropriate

### Color Contrast
- All text meets WCAG AA minimum (4.5:1 for normal text, 3:1 for large text)
- Status indicators not conveyed by color alone (icon + text)
- Links distinguished by underline in addition to color

### Motion
- Animations can be disabled via `prefers-reduced-motion`
- Typing indicator, progress bars still visible but not animated if disabled
- Focus indicators have animation but can be static

### Text
- Font sizes minimum 14px for body text
- Line height minimum 1.5
- Max line length 75 characters for readability
- Clear, plain language for instructions

## Mobile-Specific Considerations

### Responsive Breakpoints

**Small Mobile (< 480px):**
- Single column
- Full-width elements with 8px padding
- Stacked buttons (full width)
- Bottom navigation (if applicable)

**Mobile (480px - 768px):**
- Single column with better padding
- Card-based layout
- Tabbed interfaces for complex sections
- Touch targets minimum 44x44px

**Tablet (768px - 1024px):**
- Two-column layouts where appropriate
- Sidebar navigation possible
- More spacious padding

**Desktop (1024px+):**
- Full multi-column layouts
- Sidebar navigation
- Optimal spacing and margins

### Touch Interactions
- Minimum touch target: 44x44px
- Sufficient spacing between interactive elements (minimum 8px)
- Swipe gestures clearly indicated
- Long-press for additional options (with alternative method available)
