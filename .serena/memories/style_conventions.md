# Code Style and Conventions

- **Language**: Strict TypeScript. Avoid `any`.
- **Naming**: CamelCase for variables/functions, PascalCase for components/classes.
- **Components**: Functional components with Tailwind CSS for styling.
- **Validation**: Zod for schema validation (especially for AI tool inputs).
- **Comments**: Line-by-line comments for complex logic.
- **Persistence**: Hybrid model using Dexie on client and Prisma on server.
- **Architecture**: Vertical Slices methodology.
