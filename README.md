# Asset-Hatch

Asset-Hatch is an advanced AI-powered application that leverages the Vercel AI SDK to provide intelligent, conversational capabilities with hybrid persistence and generation infrastructure. This project integrates cutting-edge AI models for seamless chat functionality, asset management, and intelligent response generation.

## üéØ Features

- **AI-Powered Chat**: Integration with advanced AI models using Vercel AI SDK for intelligent conversations
- - **Hybrid Persistence**: Flexible data storage with support for multiple persistence layers
  - - **Intelligent Generation**: Built-in infrastructure for AI-powered content generation and processing
    - - **Modern Stack**: Built with TypeScript (73.4%), CSS, and HTML for robust, type-safe development
      - - **Agent Integration**: Support for AI agents with configurable memories and context management
        - - **Automated Workflows**: GitHub Actions integration for CI/CD and automated tasks including Gemini integration

                      ## üöÄ Getting Started

                      ### Prerequisites

                      - Node.js (v16 or higher)
                      - - npm or yarn package manager
                        - - Git
                         
                          - ### Installation
                         
                          - 1. Clone the repository:
                            2. ```bash
                               git clone https://github.com/zenchantlive/Asset-Hatch.git
                               cd Asset-Hatch
                               ```

                               2. Install dependencies:
                               3. ```bash
                                  npm install
                                  # or
                                  yarn install
                                  ```

                                  3. Set up environment variables:
                                  4. Create a `.env.local` file with your AI provider credentials and configuration settings.
                                 
                                  5. ### Running the Application
                                 
                                  6. Start the development server:
                                  7. ```bash
                                     npm run dev
                                     # or
                                     yarn dev
                                     ```

                                     The application will be available at `http://localhost:3000`
